Dashboards & Widgets

Create interactive dashboards with various widget types to visualize your test data.

Creating Dashboards

How do I create a dashboard?

UReport provides two dashboard creation modes to suit different use cases:

1

Click "Create Dashboard"

Navigate to Dashboards and click the Create button.

2

Enter Basic Information

Provide a name and description for your dashboard.

3

Choose Visibility

Select public (visible to all) or private (only you).

4

Select Product Lane Strategy

Choose how widgets will be filtered:

  • Single Product Focus — All widgets use one product lane (faster setup, consistent filtering)
  • Multi-Product Flexibility — Choose product lane per widget (cross-product analysis)
5

Add Widgets

Click the "Add Widget" button to create widgets. Choose a widget type, configure its settings, and it will be added to your dashboard.

Use inline editing to rename dashboards by clicking on titles. Dashboard layouts auto-save as you make changes.

How do I filter data in a Single Product Focus dashboard?

Single Product Focus dashboards bind all widgets to one product lane. Use the filter panel at the top of the dashboard to dynamically filter data across all widgets.

Core Filters

  • Product & Type (required)
  • Team
  • Browser
  • Device
  • Platform
  • Stage
  • Version

Advanced Filters

  • Date Range
  • Build Range
  • Status Filters
  • Relation Filters

When you apply filters, all widgets on the dashboard update simultaneously to reflect the filtered data. The current filter state is preserved in the URL, making it easy to bookmark or share specific views.

How do I share and collaborate on dashboards?

Dashboard Sharing

  • Public Dashboards — Visible to all users
  • Private Dashboards — Only accessible to creator
  • Clone Dashboards — Create personal copies

URL Sharing

  • Direct links with embedded filters
  • Shareable URLs preserve view state
  • Copy-to-clipboard functionality

Export Options

  • PDF Export for printable reports
  • Individual Widget Export
  • CSV Export for raw data

Widget Types

What widget types are available?

UReport offers 10+ widget types to visualize your test data:

Status Pie Chart

Interactive pie chart showing test execution results with dual view modes (Status/Triaged).

  • Toggle between Status and Triaged views
  • Interactive legend with click-to-filter
  • Export to PDF and CSV

Build History Bar Chart

Bar/line chart displaying build execution trends over time.

  • Switch between bar and line modes
  • Multi-query comparison
  • Configurable X/Y axis

Summary Card

Compact card showing key metrics: Build Summary, Status Summary, or Triaged Summary.

  • Key performance indicators
  • Trend indicators (up/down)
  • Color-coded severity

Treemap

Hierarchical visualization of test distribution across relations.

  • Group by teams, components, or tags
  • Proportional sizing by test counts
  • Interactive drill-down

Heatmap

Color-coded matrix showing pass rate trends across builds and relations.

  • Matrix view of builds vs. relations
  • Color intensity based on pass rates
  • Historical trend analysis

Test Analysis Table

Tabular analysis of test behavior patterns (Consistent vs Flaky tests).

  • Stable vs. Unstable analysis
  • Flakiness percentage calculations
  • Export to CSV

Top Tests Table

Ranked table of most-failing or slowest tests over a configurable time range (7, 30, or 90 days).

  • Toggle between Top Failures and Slowest Tests
  • Surface the most impactful tests in your suite

Trend Insights

Detects anomalous pass-rate drops across recent builds using Z-score analysis.

  • Highlights builds where pass rate deviated significantly
  • Shows build duration trend line

Relations Group By Bar

Stacked bar chart showing test counts and pass rates grouped by a relation attribute.

  • Group by components, teams, tags, or custom dimensions
  • Supports horizontal/vertical layout and table view

How do I compare data across multiple product lanes?

Some widgets support comparing data from different product lanes side by side. This is useful for cross-browser, cross-team, or cross-environment comparisons.

1

Open Widget Editor

Create a new widget or edit an existing one that supports multiple product lanes.

2

Expand "Multiple Product Lanes" Section

In the widget editor accordion, expand the "Multiple Product Lanes" panel.

3

Add Product Lanes

Click "Add Product Lane" to add additional data sources. Configure each product lane with different filter combinations.

4

Reorder and Manage

Use the arrow buttons to reorder product lanes. Edit or remove lanes as needed.

Comparison Examples
Cross-Browser
Add lanes for Chrome, Firefox, and Safari to compare test results across browsers
Team Performance
Add lanes for Frontend, Backend, and QA teams to compare team metrics
Environment
Add lanes for Dev, Staging, and Production to compare across environments

Not all widget types support multiple product lanes. The "Multiple Product Lanes" section only appears for widgets that support this feature (e.g., Build History).
